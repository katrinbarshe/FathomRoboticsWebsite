Product Description: CANaBUS Maximus
All-In-One Marine Autonomy Computer for Unmanned Surface Vessels
Enterprise-Grade Integrated Autonomy Platform
The CANaBUS Maximus represents the pinnacle of marine autonomy integration—a single-board computer that consolidates an entire USV autonomy stack into one rugged, production-ready package. Where previous solutions required separate devices for computation (Raspberry Pi), navigation (PixHawk Cube), and positioning (GPS/RTK module), the CANaBUS Maximus integrates all critical subsystems onto a single PCB within an IP67-rated automotive enclosure.
Built around the powerful Raspberry Pi Compute Module 5, paired with a PixHawk Cube Orange autopilot and precision RTK-capable GPS, this platform eliminates the complexity of multi-device integration while delivering professional-grade performance for commercial USV operations. The result is reduced cabling, simplified installation, improved reliability, and dramatically lower deployment time.
The CANaBUS Maximus is designed for operators who demand turnkey autonomy solutions without sacrificing flexibility. Whether running ROS2 navigation stacks, ArduPilot mission planning, or custom Fathom Companion interfaces, this integrated platform provides the computational horsepower, sensor fusion capabilities, and real-time control necessary for demanding maritime autonomy applications.
Key Highlights
Integrated Compute Module 5 Platform: Raspberry Pi Compute Module 5 provides quad-core ARM Cortex-A76 processing at 2.4GHz with up to 8GB RAM—delivering desktop-class performance for ROS2 navigation, machine learning inference, computer vision processing, and multi-sensor fusion. Native PCIe, dual HDMI, and USB 3.0 support enable direct connection to high-bandwidth peripherals.
Embedded PixHawk Cube Orange Autopilot: Full ArduPilot/PX4 autopilot integration with triple-redundant IMUs, barometric altimeter, and magnetometer. The Cube Orange provides L3-certified flight controller capabilities adapted for marine autonomous navigation—enabling waypoint following, geofencing, fail-safe return-to-home, and mission scripting.
RTK-Enabled Precision GPS: AS-RTK2B module with u-blox F9P dual-frequency (L1/L2) GNSS receiver delivers centimeter-level positioning accuracy when configured with RTK base station corrections. Supports GPS, GLONASS, Galileo, and BeiDou constellations for maximum availability and redundancy.
Triple Independent CAN Bus Interfaces: Three galvanically isolated CAN 2.0B ports (using TCAN4550 transceivers) provide simultaneous connectivity to engines, steering actuators, navigation equipment, and auxiliary systems—identical capability to the CANaBUS Mini but integrated into a unified platform.
Onboard Ethernet Switch: KSZ8863 dual-port Gigabit Ethernet switch enables daisy-chaining of IP-based sensors, cameras, and telemetry equipment directly through the CANaBUS Maximus enclosure—reducing harness complexity and simplifying network architecture.
Expandable NVMe Storage: M.2 NVMe slot supports high-speed solid-state storage for mission logging, video recording, map data, and neural network model storage. PCIe Gen 2 interface delivers up to 2GB/s throughput—critical for data-intensive autonomy applications.
Real-Time Clock with Battery Backup: CR2032-backed RTC maintains accurate timekeeping across power cycles—essential for timestamping sensor data, GPS time synchronization, and mission replay analysis.
Single-Board Integration: Replaces three separate devices (Raspberry Pi + PixHawk Cube + GPS module) with one integrated board, reducing points of failure, eliminating inter-device communication latency, and simplifying installation, maintenance, and troubleshooting.

Operating Modes & Software Stacks
The CANaBUS Maximus supports multiple software configurations to match different operational requirements and user expertise levels.
Mode 1: ROS2 Autonomy Stack (Recommended for Research & Advanced Users)
Deploy full ROS2 (Robot Operating System 2) on the Compute Module 5 for maximum flexibility and integration with the robotics ecosystem.
Key Features:
Native ROS2 Humble or Iron support on Ubuntu 22.04 LTS; custom ROS2 nodes for CAN bus devices (engines, steering, sensors); integration with Nav2 navigation stack; SLAM with LIDAR/camera; sensor fusion with robot_localization; MoveIt integration; RViz visualization and Gazebo simulation; access to ROS2 packages.
Typical Use Cases:
Academic research projects; multi-sensor fusion experiments; computer vision and AI/ML development; advanced perception systems; custom autonomy behaviors requiring full ROS2 flexibility.
Architecture:
CM5 (ROS2 Humble) ↔ PixHawk Cube (ArduPilot via MAVLink) → CAN Bus Nodes → Nav2 Stack → RTK GPS.
Mode 2: ArduPilot/Mission Planner (Best for Waypoint Navigation)
Run ArduRover or ArduBoat firmware on the PixHawk Cube Orange, using the CM5 as a companion computer for telemetry, video streaming, and payload management.
Key Features:
Proven ArduPilot autonomy algorithms; Mission Planner / QGroundControl integration; MAVLink telemetry; geofencing; automated survey patterns; conditional commands; companion software on CM5 for advanced features.
Typical Use Cases:
Survey and mapping missions; patrol routes; search patterns; aquaculture inspection; harbor security patrol routes.
Architecture:
PixHawk Cube (ArduRover) → Autopilot Control; CM5 (Companion) → Video Streaming, Telemetry Relay; CAN Bus → Engine/Steering via CANaBUS translation layer.
Mode 3: Fathom Companion Web Interface (Recommended for Commercial Operations)
Deploy the KiloWebInterface on the CM5 for a turnkey, zero-install remote operation platform accessible from any web browser.
Key Features:
Professional web-based control dashboard; real-time telemetry display; interactive map; sub-100ms command latency; customizable widget-based interface; multi-protocol device support; sensor fusion with data logging; event-driven automation; AI/ML integration; fleet management; mission replay; video streaming integration.
Typical Use Cases:
Shore-based remote piloting; multi-vessel fleet operations; custom mission profiles; enterprise system integrations; demonstrations and trials.
Architecture:
CM5 (KiloWebInterface Server) → Drivers for CAN/Serial/GPIO → Web UI → Remote Operator.
GitHub:
github.com/isensystech/KiloWebInterface
Mode 4: Hybrid Architecture (Advanced Integration)
Combine multiple modes for maximum capability—for example, running ROS2 on the CM5 while leveraging the PixHawk Cube's autopilot algorithms. PixHawk Cube handles stabilization and waypoint navigation; CM5 runs high-level mission planner, AI, and sensor fusion.
System Integration Benefits
Before CANaBUS Maximus: Multi-Device Complexity
Traditional USV autonomy stacks require multiple separate devices (compute, autopilot, GPS, CAN interfaces), creating complex wiring, multiple power domains, latency, and maintenance overhead.
Problems with Traditional Approach:
Complex wiring; lengthy installation; multiple failure points; serial communication latency; difficult troubleshooting; large space requirements; vibration sensitivity; configuration drift.
After CANaBUS Maximus: Single-Board Simplicity
Integrated CANaBUS Maximus consolidates compute, autopilot, GPS, and CAN/Ethernet backbones into one enclosure, simplifying installation, reducing cables, improving reliability, and lowering latency.
Advantages of Integrated Approach:
Single enclosure and power input; fewer cables; higher reliability; sub-5ms on-board communication; simpler diagnostics; compact form factor; vibration resistance; unified configuration.

Getting Started
Quick Start Guide
Hardware Installation: Mount CANaBUS Maximus; connect 12V DC; connect CAN buses; connect Ethernet; mount GPS antenna; connect RTK correction source.
Initial Configuration: Insert pre-loaded MicroSD; power on; access configuration; set network; load DBC files; calibrate compass/IMU; set home and geofence.
Software Selection: ROS2 (Ubuntu 22.04 + ROS2 Humble), ArduPilot companion stack, or Fathom Companion (KiloWebInterface).
Testing & Validation: Shore-based checks, GPS/RTK verification, remote control tests, dockside trials, open-water trials.

Maximus: all-in-one autonomy computer (CM5, PixHawk Cube, RTK GPS, triple CAN, Ethernet switch, NVMe). Mini: external companion device focused on CAN integration. Choose Mini for retrofit/CAN translation; choose Maximus for new builds and full integration.
