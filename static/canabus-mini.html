Universal CANaBUS Controller for Unmanned Systems
Can Controller for Industrial & Commercial USV Applications The CANaBUS Mini is a professional-grade autonomy controller designed to transform conventional crewed vessels into sophisticated Vehicles (USVs). By providing seamless integration with CAN bus systems over Ethernet. The CANaBUS Mini bridges the gap between legacy electronics and modern autonomous control stacks like ROS/ROS2, Ardupilot or Fathom Robotics USV Companion Interface.
Housed in a rugged IP69K automotive-grade enclosure, this compact controller manages critical vehicle systems including engines (ECU), steering actuators, trim tabs, autopilots, through three independent CAN bus interfaces. Combined with dual UART serial ports and Ethernet connectivity, the CANaBUS Mini serves as command normalizer and interpreter system for unmanned operations.
Built on the high-performance ARM Cortex-M7 microcontroller platform, the CANaBUS Mini delivers deterministic, real-time control with minimal latency — essential for sensor fusion and autonomous decision-making in dynamic environments. Whether deployed as a standalone CAN translator, integrated into ROS2 autonomy stacks, or paired with established autopilot systems, the CANaBUS Mini provides the reliable, industrial-strength foundation required for commercial uncrewed systems operations.

Key Highlights
Multi-Protocol Marine Integration:
Interfaces with proprietary CAN protocols from major manufacturers like Volvo Penta, Yamaha, Mercury, Garmin, and Raymarine.
Triple Independent CAN Architecture:
Features three isolated CAN 2.0B interfaces for simultaneous, conflict-free connection to engine, steering, and navigation subsystems.
Industrial-Grade Build:
IP69K-rated automotive enclosure with marine-grade connectors and conformal-coated PCBs for harsh saltwater protection.
Real-Time Processing:
Powered by a 600MHz ARM Cortex-M7 delivering <5ms latency, essential for safe autonomous response.
Flexible Operating Modes:
Supports four modes ranging from simple protocol translation via DBC files to full ROS2 autonomy integration.
Remote Web Interface:
Native compatibility with the Fathom Robotics USV Companion Interface for real-time telemetry, diagnostics, and piloting from any browser.
Core Operating Modes
MODE 1:
Standalone CAN Transceiver & Protocol Translator Functions as an intelligent gateway, converting proprietary protocols into NMEA 2000 or JSON via Ethernet/Serial using proprietary .dbc files.
• Use Cases:
Retrofitting legacy control elements and electronics into modern remotely controlled or autonomous systems.
• Key Features:
DBC file-based parsing, configurable routing, <5ms latency, and control system agnostic.
MODE 2:
ROS/ROS2 Integration Node Acts as a dedicated hardware node within ROS architectures to expose marine CAN devices as native ROS entities.
• Use Cases:
Academic autonomy research, multi-sensor fusion, and computer vision/LIDAR integration.
• Key Features:
Custom device-specific ROS2 nodes, bidirectional sensor/control data flow, and Nav2/MoveIt compatibility.
MODE 3:
Autopilot Integration (ArduPilot/QGC) Connects industrial CAN hardware to open-source platforms like ArduPilot and QGroundControl.
• Use Cases:
Utilizing proven autonomy algorithms, waypoint mission planning, and MAVLink telemetry integration.
• Key Features:
MAVLink bridge for CAN, custom ArduPilot drivers, and support for ArduRover/ArduBoat firmware.
MODE 4:
Fathom Robotics USV Companion Interface (Recommended) The most complete mode, pairing the device with the KiloWebInterface dashboard for unmanned vessel operations.
• Use Cases:
Professional commercial USV operations, remote shore-based piloting, and fleet management (coming soon).
• Key Features:
Zero-install web interface, real-time telemetry with custom widgets, <100ms command latency, mission replay, and modular sensor fusion. Includes advanced payload release capabilities extending beyond the standard ArduPilot scope.
